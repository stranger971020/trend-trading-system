## 一、项目概述

### 1.1 目标

构建一套**全自动**的 A 股趋势交易分析系统，每日完成以下闭环：

```
数据获取 → 多维度分析 → 报告生成 → Telegram 推送 + GitHub Pages 展示
```

### 1.2 核心理念

**"三级领先 → 一级确认 → 个股精选"**

- 从 258 个申万**三级**行业中捕捉微观领先信号
- 用 31 个申万**一级**行业确认板块趋势持续性
- 在趋势行业中精选个股，搭配多维度风控

| 数据规模 | 91 万条记录 · 3 类资产 |

### 3.2 核心算法

#### 板块持续性评分（M2）

```
持续性得分 = 动量分×0.30 + 收益斜率×0.25 + 换手率×0.20 + 相对强度×0.25

动量分 = RSI(14)分×0.25 + MACD金叉/死叉分×0.25 + 布林位置分×0.25 + 趋势偏离分×0.25
```

- 高持续性 ≥ 7.0 | 中等持续性 ≥ 5.0 | 低持续性 < 5.0

#### 个股评分（M3）

```
个股得分 = 超额收益(20日)×50% + 5日动量×30% + MA20偏离度×20%

超额收益 = 个股20日收益 - 所属行业20日收益
```

#### MACD 顶背离检测（M1）

```
60日窗口内扫描局部价格峰：
  IF 最近价格峰 > 前次价格峰 AND 最近DIF峰 < 前次DIF峰
  → 顶背离预警
```

#### 宏观状态机（R1）

```
ADX(14) ≥ 25 AND Price > MA200 → BULL (7-8成)
ADX(14) ≥ 25 AND Price < MA200 → BEAR (2-3成)  
ADX(14) < 25                    → RANGE (5成)
```

## 六、每日任务规划

### 6.1 调度时间线（北京时间）

```
20:00   cron 触发 run_analysis.py
20:00   检查是否为交易日（Tushare trade_cal）
20:00   增量更新 L1 行业日线（31个，akshare，~12秒）
20:01   增量更新 L3 行业日线（258个，akshare，~80秒）
20:02   增量更新个股日线（按需拉取，Tushare）
20:03   拉取最新资金流+融资数据（Tushare）
20:04   执行 12 个分析模块（~5秒）
20:04   生成 HTML 报告 → reports/
20:04   生成文字报告 → Telegram 推送
20:05   更新虚拟持仓记录
20:05   完成（总耗时 ~4分钟）
```
